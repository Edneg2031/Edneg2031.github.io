为什么图像可以分块： ViT

1 问题背景

Transformer 原本处理的是文本 token，比如：

我 / 喜欢 / 学习 / 多模态

但是图像本身不是一串离散 token，而是一个二维像素矩阵：

\[ \mathbf{H\times W \times C} \] 所以问题是：图像能不能也变成一串 token？ViT 和 DiT 都给出了类似的回答：把图像切成 patch，每个 patch 当作一个 token。 \[ \mathbf{[Batch, Num\_tokens, Embedding\_size]} \]

2 ViT

  1. 输入图像: \(H × W × C\) （\(224 \times 224 \times 3\)）
  2. 切patch块，每块的形状为：\(P × P × C\) ($ 16 $)，可以切成 \(patch\_size\times patch\_size(14 \times 14=196)\)个 patch 块，
  3. 把 patch 块通过 Linear Projection 映射成 token embedding \((patch\_size \times patch\_size, embed\_size)\)
  4. 加入 position embedding
  5. 送入 Transformer Encoder

2.1 Patch Embedding

工程上常见的实现是用卷积直接完成 分块 和 Linear Projection 的操作， - 输入图像形状：\(\mathbf{[Batch, H, W, C]}\) - Patch Embedding过后形状： \(\mathbf{[Batch, patch\_size \times patch\_size, embed\_size]}\) 记为 \(\mathbf{[Batch, N, embed\_size]}\)

得到 \(\mathbf{[Batch, N, embed\_size]}\) 后 还会添加一个 [CLS] Token 来作为分类的Token， 形状为 \(\mathbf{[Batch, 1, embed\_size]}\)，在后续进行 attention 操作的时候会综合每个 patch token 的信息。

2.3 Positional Embedding

因为 Transformer 结构没有感知空间的能力，所以需要添加 Positional Embedding，ViT 论文中用的是Learnable Positional Embedding的方式来添加。设置一个可以先学习的参数 $ E^{i}_{pos} $， 其维度和与 patch tokens 一致可以直接进行相加。
