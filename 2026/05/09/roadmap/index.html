多模态大模型学习 Roadmap

Executive Summary

这份 Roadmap 采用一条尽量稳、尽量少走弯路的主线：先把图像为什么能被 patchify 讲清楚，再理解图文对齐和视觉接入 LLM 的方式，最后回到 urlQwen-VL 系列官方资源turn2search1 / urlQwen2-VL 论文turn3search5 / urlQwen2.5-VL 技术报告turn1search1 做系统精读与本地实践。这个顺序与当前开源视觉语言模型的技术演进基本一致：urlViT 论文turn0search0 解决“图像如何变 token”，urlDiT 论文turn0search1 把 Transformer 推进到扩散模型，urlCLIP 论文turn0search2 解决“图文如何对齐”，urlBLIP-2 论文turn0search3 和 urlLLaVA 论文turn1search0 则展示了视觉特征如何高效接入 LLM。citeturn8view0turn8view1turn0search2turn9view1turn8view4turn9view2turn3search1

按你给定的投入强度，每周 10–15 小时、连续 12 周，是足够走完“论文读懂 + 小实验 + Hexo 系列笔记 + 一个可放简历的小项目”这一闭环的；真正需要避免的不是时间不够，而是过早分散到 RL、Agent、分布式、Serving 全栈，结果每个方向都只停留在关键词级别。官方中文/官方教程体系里，最适合作为骨架的是 urlQwen 中文文档turn4search2、urlQwen 快速开始turn6search3、urlCommunity Computer Vision Courseturn1search6、urlVLM 入门课程turn1search18、urlSmol Course 的 VLM 单元turn11search4，以及来自 entity[“organization”,”Stanford University”,”Stanford, CA, USA”] 的 urlCS231n 课程与讲义turn1search7。citeturn12view0turn12view1turn9view7turn12view6turn1search7turn18search0turn18search7turn18search8

对 3060 笔记本而言，最重要的策略不是“硬跑 7B/72B 全流程”，而是把“完整训练”降级为“量化推理 + 小样本比较 + 微型 SFT/LoRA”；官方的 urlQwen2-VL-7B TRL recipeturn15search2 明确提示这是 resource-intensive 并需要 A100，而 urlSmolVLM consumer GPU recipeturn15search1、urlSmolVLM-256M-Instructturn19search2 和 urlSmolVLM2-2.2B-Instructturn19search1 更适合作为本地微调与 sanity check 起点。量化层面，urlTransformers Quantization 文档turn6search18 和 urlbitsandbytes 文档turn6search10 提供了 4-bit/8-bit 的正式路径。citeturn15search2turn15search1turn19search2turn19search1turn6search18turn6search10

路线设计原则

把学习主线压成“patchify → alignment → bridge → instruction tuning → Qwen-VL 演进 → 轻量实践/评测”，比直接“泛学多模态”更稳妥。原因是：ViT 把图像直接视为 patch token 序列；DiT 延续 patch 思路到 latent diffusion；CLIP 让图像和文本进入共享语义空间；BLIP-2 用 Q-Former 把冻结视觉编码器和冻结 LLM 连接起来；LLaVA 则把视觉接入方式和 instruction tuning 结合起来；到 Qwen2-VL / Qwen2.5-VL，这条链路又扩展到了动态分辨率、视频、文档解析和 grounding。citeturn8view0turn8view1turn0search2turn9view1turn8view4turn9view2turn3search1

下面两张示意图，是你整个 12 周里最该反复重画、反复解释的两张图。能把它们讲清楚，说明你已经真正进入主线，而不是停留在“知道模型名字”的层面。citeturn24search1turn8view1turn9view7turn9view1turn8view4turn3search1

ViT 的关键信息其实非常凝练：图像被切成固定大小 patch，patch 被当成 token 序列送入纯 Transformer；官方论文和文档都强调这一步是 Vision Transformer 的核心入口。citeturn8view0turn24search1turn18search2

VLM 的通用结构也同样清晰：视觉编码器先产生视觉 token，再通过 projector 或 Q-Former 之类的桥接层把视觉特征送入语言模型。urlVLM 入门课程turn1search18 把这件事讲得很系统；urlBLIP-2 论文turn0search3 给出 Q-Former 两阶段预训练；urlLLaVA 论文turn1search0 展示 visual instruction tuning；urlQwen2.5-VL 技术报告turn1search1 则把这一框架进一步扩展到了 grounding、文档、视频与 visual agent。citeturn9view7turn9view1turn8view4turn3search1

这份路线默认你每周可稳定投入 10–15 小时。建议每周的配比固定为“阅读 5–6 小时 + 画图/整理 2–3 小时 + 代码实验 3–4 小时 + Hexo 笔记 2 小时”，这样就能同时保住理解深度和可执行性。你的终点不是“看完论文”，而是满足四个标准：能白板解释两张核心图、能跑通至少两个本地 demo、能做出一个带误差分析的小项目、能把整个过程写成一组可检索的 Hexo 文章。citeturn12view6turn12view7turn12view1turn8view6turn8view7
