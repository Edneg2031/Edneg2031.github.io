Hexo + Butterfly 数学公式显示问题总结

在 Hexo 的 Butterfly 主题中，数学公式显示异常通常不是 Butterfly 本身的问题，而是：

Markdown 渲染器先处理了文章内容，导致 LaTeX 公式在被 MathJax 渲染之前就已经被破坏。

如果 Markdown renderer 在 MathJax 之前把 _、\、$ 等内容处理乱了，那么 MathJax 后面就无法正确识别公式。

所以问题不是公式本身错，而是公式在进入 MathJax 之前被 Markdown 渲染器破坏了。

网上有些方法说，在所有下划线前加反斜杠：

这种方法不稳定。

有些环境中，Markdown renderer 会先把 \_ 还原成 _，然后 MathJax 能正常渲染。

但有些环境中，反斜杠会被保留下来，结果页面会直接显示反斜杠，导致公式仍然异常。

所以不推荐长期使用这种方式。

10. 最终经验总结

Hexo + Butterfly 公式异常的关键点是：

Butterfly 负责加载 MathJax，但公式在进入 MathJax 之前，会先经过 Hexo 的 Markdown renderer。

因此，公式显示失败通常有三类原因：

  1. Markdown renderer 把行内公式里的 _、\、$ 处理坏了。
  2. 公式写法本身不规范，比如 \bar{\alpha_t}、\rarr。
  3. 复制粘贴带入了不可见字符。
